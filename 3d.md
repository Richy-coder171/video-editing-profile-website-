# 3D Model System

This project uses lightweight procedural Three.js models to add a premium cinematic feel without shipping heavy 3D asset files. The models are built directly in React components using Three.js geometry, materials, lights, and animation loops.

## Main Hero Model

The homepage hero uses `client/src/components/sections/Hero3DModel.jsx`.

This model is a cinematic editing workstation scene. It includes:

- A glowing program monitor
- Timeline clip bars
- Camera lens geometry
- A clapperboard
- Floating UI panels
- Blue, amber, violet, and ember neon accents
- Subtle pointer-based rotation
- Reduced-motion support

The model is lazy-loaded from `HeroSection.jsx` so the main page can render first and the 3D scene loads only when needed.

## Page Accent Models

The rest of the site uses a shared reusable 3D system in:

- `client/src/components/three/LazyPage3DAccent.jsx`
- `client/src/components/three/Page3DAccent.jsx`

`LazyPage3DAccent.jsx` lazy-loads the page model renderer with React `Suspense`.

`Page3DAccent.jsx` creates different procedural 3D variants depending on the page:

- `reels`: vertical phone/reel model with glowing 9:16 style panels
- `videos`: widescreen monitor, camera lens, and timeline model
- `designs`: layered poster/card stack for Photoshop, Illustrator, thumbnails, posters, logos, and social designs
- `about`: clapperboard and floating creative tool panels
- `contact`: message/envelope model for project booking
- `detail`: project-detail media monitor scene
- `admin`: secure upload/dashboard block model
- `notFound`: broken-panel 404 scene

## Pages Using 3D Accents

3D accents are used on:

- Home
- Reels
- Videos
- Designs
- About
- Contact
- Project detail
- Admin login
- Admin dashboard
- 404 page

The models are positioned as background or edge accents so they support the layout without blocking portfolio content, forms, buttons, or uploaded media.

## Performance Notes

- Three.js is split into its own Vite chunk named `three-vendor`.
- Page models are lazy-loaded through `React.lazy`.
- The shared page accent chunk is small because it reuses the existing Three.js vendor chunk.
- Models are procedural, so there are no `.glb`, `.gltf`, texture, or video model assets to download.
- Pixel ratio is capped to reduce GPU cost.
- Animations respect `prefers-reduced-motion`.
- WebGL contexts and geometries/materials are disposed on component unmount.

## Visual Style

The 3D direction follows the website theme:

- Dark cinematic surfaces
- Neon blue, amber, ember, and violet highlights
- Anime-inspired editing energy without a childish look
- Manga-panel and studio-monitor visual language
- Professional portfolio presentation for video editing and graphic design work

The backend, Cloudinary upload flow, Supabase metadata system, authentication, and protected routes are not changed by the 3D model system.
